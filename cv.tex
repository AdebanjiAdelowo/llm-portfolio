%---------- PROJECTS ----------
\section{Selected Projects}

\noindent
\textbf{\href{https://github.com/adebanjiadelowo/llm-portfolio/tree/main/03-bank-customer-risk-engine}{Bank Customer Risk Engine}}:
Embedding-powered risk scoring system for retail banking: encodes customer profiles into 384-dim sentence-transformer vectors, trains a multi-output model to simultaneously predict loan approval probability, default risk, and interest rate; cosine-similarity search enables nearest-neighbour profiling.
\textit{Stack: sentence-transformers, XGBoost, scikit-learn, UMAP, Python}

\textbf{\href{https://github.com/adebanjiadelowo/llm-portfolio/tree/main/02-enterprise-nl2sql}{Enterprise NL2SQL Pipeline}}:
Two-stage LLM pipeline converting natural language to SQL over an 8-table enterprise database, cutting prompt token cost by \textbf{$\sim$60\%}. Complexity router auto-escalates multi-table queries; includes retry logic and SQL injection guard.
\textit{Stack: OpenAI API (GPT-4o-mini), Python, pandas, SQLite}

\textbf{\href{https://github.com/adebanjiadelowo/llm-portfolio/tree/main/01-financial-sentiment-distillation}{Financial Sentiment Distillation}}:
Compressed FinBERT by 62\% (110M $\to$ 88M parameters) via Taylor-gradient pruning and knowledge distillation, improving accuracy from 84.5\% to \textbf{96.9\%} on the financial\_phrasebank benchmark.
\textit{Stack: PyTorch, HuggingFace Transformers, scikit-learn}

\textbf{\href{https://github.com/adebanjiadelowo/llm-portfolio/tree/main/04-data-analyst-agent}{LLM-Powered Data Analyst Agent}}: Autonomous data analysis agent using LangChain's Pandas DataFrame Agent; accepts natural-language instructions to clean data, identify correlations, generate visualisations, and run forecasting models (ARIMA) against a real climate dataset.
\textit{Stack: LangChain, OpenAI GPT, Pandas, Python}

\textbf{\href{https://github.com/adebanjiadelowo/llm-portfolio/tree/main/05-medical-rag-assistant}{Medical RAG Assistant}}: Conversational medical assistant over the MedQuad Q\&A knowledge base; ChromaDB stores embeddings, a LangChain ReAct agent routes medical vs.\ general queries, and ConversationBufferWindowMemory maintains context across turns.
\textit{Stack: LangChain, ChromaDB, OpenAI, HuggingFace}

\textbf{\href{https://github.com/adebanjiadelowo/llm-portfolio/tree/main/06-llm-evaluation-langsmith}{LLM Evaluation with LangSmith}}: Systematic evaluation framework comparing T5-base, fine-tuned T5, and OpenAI GPT on CNN/DailyMail summarisation using LangSmith tracing and cosine embedding distance; shows domain fine-tuning closes most of the gap with proprietary models.
\textit{Stack: LangSmith, LangChain, HuggingFace T5, OpenAI}

\textbf{\href{https://github.com/adebanjiadelowo/llm-portfolio/tree/main/07-lora-qlora-finetuning}{LoRA and QLoRA Fine-Tuning}}: Parameter-efficient fine-tuning of Llama-3.2-1B (LoRA) and Llama-3-8B (QLoRA with 4-bit NF4 quantisation); trains less than 1\% of model weights, enabling 8B-parameter fine-tuning on a single 16\,GB GPU.
\textit{Stack: PEFT, BitsAndBytes, HuggingFace Transformers, trl}
